# Istruzioni per contributori

Queste linee guida si applicano all'intero repository.

- Mantieni la documentazione in italiano, preferendo esempi pratici e sezioni brevi.
- Quando modifichi codice o configurazioni, aggiorna anche la documentazione correlata.
- Per i commit, usa messaggi descrittivi in formato "categoria: descrizione" (es. "docs: aggiorna README").
- Dopo aver eseguito test o build, riporta i comandi eseguiti nel messaggio finale.
- Non aggiungere blocchi `try/catch` intorno agli import nelle basi di codice esistenti.
- Ricorda di eseguire i lint/test disponibili prima di proporre modifiche che toccano il runtime.
